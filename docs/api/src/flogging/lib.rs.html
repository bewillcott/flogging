26//!
27//! # FLogging
28//!
29//! The primary purpose of logging, is to facilitate fault diagnosis through the
30//! provision of specific information as, when, and from where, it is needed. This could
31//! be during development, testing, or even during production runs.
32//!
33//! There is a new tutorial guide: **[The FLogging Guide]**.
34//!
35//! ## Setting up
36//!
37//! You need to add this crate to your project:
38//! ```text
39//! $ cargo add flogging
40//! ```
41//! or add this text to the projects `Cargo.toml` file:
42//! ```text
43//! [dependencies]
44//! flogging = "0.6.0"
45//! ```
46//! ## ** Warning **
47//!
48//! Before proceeding, please read the [README.md] file.
49//!
50//! ## Features
51//!
52//! - [Levels](enum.Level.html) - There are nine (9) levels of message logging, with two (2) special ones.
53//! - [Choice](index.html#choice) - You can use either macros, methods, or a mix of both.
54//! - [Built-in options](index.html#built-in-options) - A range of handlers and formatters.
55//! - [Customization](index.html#customization) - You can create your own handlers and/or formatters.
56//!
57//! ### Choice
58//!
59//! - [Macros](index.html#macros)
60//! - [Methods](index.html#methods)
61//!
62//! #### Macros
63//!
64//! - [Special Note](index.html#special-note)
65//!
66//! This crate has very easy to use macros. By using them, you remove a lot of the complexity
67//! from the process. Thus making it both simpler and less code cluttering, to use.
68//!
69//! - There is one macro ([`const_logger!()`]) that is used to setup a single module/file for logging.
70//! - There is one macro ([`#[logger]`][macro@logger]) that is applied as an attribute to each function/method
71//!   that you need to create log entries within.
72//! - There are nine macros that are used to actually log the messages:
73//!     - [`config!()`]
74//!     - [`entering!()`]
75//!     - [`exiting!()`]
76//!     - [`fine!()`]
77//!     - [`finer!()`]
78//!     - [`finest!()`]
79//!     - [`info!()`]
80//!     - [`severe!()`]
81//!     - [`warning!()`]
82//! - There are two helper macros:
83//!     - [`get_handler!()`]
84//!     - [`set_level!()`]
85//!
86//! Check out the [Examples](index.html#examples) below, or **[The FLogging Guide]**, for how easy it is to get started.
87//!
88//! ##### Special Note
89//!
90//! For the macros that accept the parameter: `msg`, the following is true:
91//!
92//! - They accept parameters the same as for [`std::format!`](https://doc.rust-lang.org/std/macro.format.html)
93//!     - plain text `&str`: `("It's your time.")`
94//!     - format `&str` with interpolated variables: `("Var: {var}")`
95//!     - format `&str` with supporting parameters: `("Var: {}", var)`
96//!     - Combination of the last two: `("Vars {var1} - {}:{}", var2, var3)`
97//! - Additional Feature
98//!     - Just one or more variables without a supplied format string: `(var1, var2, var3)`
99//!     - In this case, a default format string will be used: `"{}, {}, {}"`
100//!     - The number of `"{}"` will depend on the number of parameters.
101//!     - Ideal for logging concrete instances that have very good `Display` implementations,
102//!       or you just need their data without further explanation.
103//! - Special Cases
104//!     - [entering!] and [exiting!]
105//!     - These two macros have the same features as the others,
106//!       but they may also be used _without_ any parameters. In such
107//!       a case, their defaults will be used.
108//!
109//! #### Methods
110//!
111//! - [Logger](index.html#logger)
112//! - [LoggerBuilder](index.html#loggerbuilder)
113//!
114//! Now for the coding geeks! Yes I didn't forget you lot.
115//!
116//! Though the macros are the easiest and simplest way to use this crate, those macros are just candy
117//! coating over the real workers, the methods. There are two main mods/structs in this crate, [`Logger`](struct.Logger.html)
118//! and [`LoggerBuilder`](struct.LoggerBuilder.html).
119//!
120//! ##### Logger
121//!
122//! `Logger` is the work-horse of the crate. It has all the methods for initializing each function/method
123//! for logging, and all of the message logging methods.
124//!
125//! Using the "methods" option is more complex, as-in, you have to write a lot more code, and manage it.
126//! To see how much more is involved, check-out the [`Logger`](struct.Logger.html)'s methods. There are
127//! plenty of examples throughout.
128//!
129//! - [`builder()`][Logger::builder]
130//! - [`config()`][Logger::config]
131//! - [`console_logger()`][Logger::console_logger]
132//! - [`econsole_logger()`][Logger::econsole_logger]
133//! - [`custom_logger()`][Logger::custom_logger]
134//! - [`entering()`][Logger::entering]
135//! - [`entering_with()`][Logger::entering_with]
136//! - [`exiting()`][Logger::exiting]
137//! - [`exiting_with()`][Logger::exiting_with]
138//! - [`file_logger()`][Logger::file_logger]
139//! - [`fine()`][Logger::fine]
140//! - [`finer()`][Logger::finer]
141//! - [`finest()`][Logger::finest]
142//! - [`fn_name()`][Logger::fn_name]
143//! - [`get_handler()`][Logger::get_handler]
144//! - [`has_handler()`][Logger::has_handler]
145//! - [`info()`][Logger::info]
146//! - [`level()`][Logger::level]
147//! - [`set_fn_name()`][Logger::set_fn_name]
148//! - [`set_level()`][Logger::set_level]
149//! - [`severe()`][Logger::severe]
150//! - [`string_logger()`][Logger::string_logger]
151//! - [`warning()`][Logger::warning]
152//!
153//! ##### LoggerBuilder
154//!
155//! `LoggerBuilder` is used by `Logger` to provide various configuration options for setting up your logger.
156//! The available options/methods are:
157//!
158//! - [`add_console_handler()`][LoggerBuilder::add_console_handler()]
159//! - [`add_console_handler_with()`][LoggerBuilder::add_console_handler_with()]
160//! - [`add_econsole_handler()`][LoggerBuilder::add_econsole_handler()]
161//! - [`add_econsole_handler_with()`][LoggerBuilder::add_econsole_handler_with()]
162//! - [`add_console_handler()`][LoggerBuilder::add_pconsole_handler()]
163//! - [`add_console_handler_with()`][LoggerBuilder::add_pconsole_handler_with()]
164//! - [`add_custom_handler()`][LoggerBuilder::add_custom_handler()]
165//! - [`add_custom_handler_with()`][LoggerBuilder::add_custom_handler_with()]
166//! - [`add_file_handler()`][LoggerBuilder::add_file_handler()]
167//! - [`add_file_handler_with()`][LoggerBuilder::add_file_handler_with()]
168//! - [`add_string_handler()`][LoggerBuilder::add_string_handler()]
169//! - [`add_string_handler_with()`][LoggerBuilder::add_string_handler_with()]
170//! - [`remove_file()`][LoggerBuilder::remove_file()]
171//! - [`set_fn_name()`][LoggerBuilder::set_fn_name()]
172//! - [`set_level()`][LoggerBuilder::set_level()]
173//!
174//! And to finish:
175//! - [`build()`][LoggerBuilder::build()]
176//!
177//! These options/methods allow you a lot of flexibility in how you configure your logger. As you will typically
178//! have a different logger for each mod/file, you have a lot of control over what is logged, how it is formatted,
179//! and where it is stored/viewed. With the [`set_level()`][LoggerBuilder::set_level()] method, you can control
180//! this on a mod/file basis. Logging each mod/file differently, or even turning logging off when you no-longer require it.
181//!
182//! Check out **[The FLogging Guide]** for examples and further help.
183//!
184//! **Note**
185//!
186//! As of version (0.4.0), you can only set the logging level for the logger. All handlers process every log entry
187//! that the logger accepts, based on the logger's current log level setting. This may change in a future version,
188//! allowing each handler to have its own logging level.
189//!
190//! ### Built-in options
191//!
192//! I have included a number of handlers to get you started:
193//!
194//! - [`ConsoleHandler`]
195//!   - [`ConsoleType::StdOut`]
196//!   - [`ConsoleType::StdErr`]
197//!   - [`ConsoleType::Production`]
198//! - [`FileHandler`]
199//! - [`MockHandler`]
200//! - [`StringHandler`]

202//! There are also a number of formatters as well:
203//!
204//! - [`Iso8601Formatter`]
205//! - [`MockFormatter`]
206//! - [`SimpleFormatter`]
207//! - [`UnixTimestampFormatter`]

209//! ### Customization
210//!
211//! - [Custom Handler](index.html#custom-handler)
212//! - [Custom Formatter](index.html#custom-formatter)
213//!
214//! Now for the fun part - "Doing it _your_ way!!!"
215//!
216//! Though I have provided some "standard" handlers and formatters, not everyone, or every project,
217//! will want to use them. I expect there will be a need for:
218//!
219//! - sending log entries to remote system log servers,
220//! - sending log entries to another program (local or remote) for live analysis, or some other processing,
221//! - storing log entries in a specific file format (xml, json, csv),
222//! - storing log entries in a database.
223//!
224//! And I'm sure you'll come-up with more requirements at some time in the future. So, you have the option
225//! to create your own custom handlers and custom formatters. Mixing them up with the built-in ones as
226//! you need to.
227//!
228//! OK now, how do you do it. Well this is going to require some work on your part.
229//!
230//! ---
231//!
232//! For a more in-depth tutorial refer to **[The FLogging Guide]**.
233//!
234//! ---
235//!
236//! #### Custom Handler
237//!
238//! To create a custom handler, I would suggest looking at the source code for the built-in ones, and copying
239//! the code from the one that is closest to your requirements. _Make sure that you rename as appropriate!_
240//! Then make the necessary changes, adding in your own code, to get it doing what you need.
241//!
242//! When you are ready to try-out your new custom handler, check-out these methods:
243//!
244//! - [`Logger::custom_logger()`]
245//! - [`LoggerBuilder::add_custom_handler()`]
246//! - [`LoggerBuilder::add_custom_handler_with()`]
247//!
248//! #### Custom Formatter
249//!
250//! Now for the custom formatter. This may require a bit more investigation on your part, as to the actual
251//! formatting options that are available.
252//!
253//! Firstly, this crate uses [crono] for the date/time functionality. Check
254//! out the available [specifiers]. You will need to use the formatting options from this crate for the `dt_fmt` string, of your custom
255//! formatter.
256//!
257//! Secondly, the `fmt_string` uses the format options available in accordance with [std::fmt]. Though I am
258//! actually using the [strfmt] crate to do the formatting, because it does
259//! _not_ require a 'static' string like `format!()`.
260//!
261//! Again, check-out the built-in formatters, and copy the code from the one that is closest to your
262//! requirements. _As before, renaming as necessary!_ Also, check-out the trait: [`FormatTrait`](trait.FormatTrait.html).
263//! You will need to implement it for your custom formatter, as you will notice when you look at the built-in formatters.
264//! Also, you will find that the 'provided method', [`ft_fmt()`][FormatTrait::ft_fmt], provides certain variables
265//! that you can include, via interpolation, in your `fmt_string`.
266//!
267//! Once you have got your custom formatter set up, you can then use it with:
268//!
269//! - [`LoggerBuilder::add_console_handler_with()`]
270//! - [`LoggerBuilder::add_econsole_handler_with()`]
271//! - [`LoggerBuilder::add_pconsole_handler_with()`]
272//! - [`LoggerBuilder::add_custom_handler_with()`]
273//! - [`LoggerBuilder::add_file_handler_with()`]
274//! - [`LoggerBuilder::add_string_handler_with()`]
275//!
276//! ## Examples
277//!
278//! This example demonstrates the use of the macros. The reason I am demoing the macros, is that I expect most
279//! people will want to use them, instead of the methods, for ease of use.
280//!
281//! Let's see what is required:
282//!
283//! 1. At the module/file level:
284//!     - `use flogging::*;`
285//!     - `const_logger!({...});`[=>][const_logger]
286//! 2. On each function/method you want to add logging to:
287//!     - `#[logger]`[=>][logger]
288//! 3. Inside each such attributed function/method:
289//!     - Any of the logging [macros]
290//!
291//! ```
292//! use flogging::*;
293//! use std::{error::Error, result::Result};
294//!
295//! // Setting up the module level logger.
296//! const_logger!({
297//!     Logger::builder(module_path!())
298//!         .add_console_handler()
299//!         .add_file_handler("test_logs/debug.log")
300//!         .set_level(Level::FINEST)
301//!         .build()
302//! });
303//!
304//! #[logger]
305//! fn do_something() {
306//!     entering!();
307//!
308//!     // do some work worth noting
309//!     let result = "Just something to log.";
310//!     info!("Did some work here.\n  {result}");
311//!
312//!     // ...
313//!
314//!     fine!("Bit more detail.");
315//!
316//!     if let Err(e) = error_prone() {
317//!         warning!("Error: {}", e);
318//!     }
319//!
320//!     exiting!();
321//! }
322//!
323//! #[logger]
324//! fn error_prone() -> Result<(), Box<dyn Error>> {
325//!     entering!();
326//!     let rtn = Err(Box::from("Bad day!"));
327//!     exiting!();
328//!     rtn
329//! }
330//!
331//! #[logger]
332//! fn main() {
333//!     entering!();
334//!     info!("All logging macros accept the same parameters as `std::format!(...)`");
335//!     warning!("Those same macros (info, etc.) MUST have atleast one parameter.");
336//!     config!("This is running on Fedora Linux 42.");
337//!     do_something();
338//!     info!("Job's done.");
339//!     exiting!();
340//! }
341//!
342//! ```
343//! Output:
344//! ```code
345//! flogging->main [FINER  ] Entry
346//! flogging->main [INFO   ] All logging macros accept the same parameters as `std::format!(...)`
347//! flogging->main [WARNING] Those same macros (info, etc.) MUST have atleast one parameter.
348//! flogging->main [CONFIG ] This is running on Fedora Linux 42.
349//! flogging->do_something [FINER  ] Entry
350//! flogging->do_something [INFO   ] Did some work here.
351//!   Just something to log.
352//! flogging->do_something [FINE   ] Bit more detail.
353//! flogging->error_prone [FINER  ] Entry
354//! flogging->error_prone [FINER  ] Return
355//! flogging->do_something [WARNING] Error: Bad day!
356//! flogging->do_something [FINER  ] Return
357//! flogging->main [INFO   ] Job's done.
358//! flogging->main [FINER  ] Return
359//! ```
